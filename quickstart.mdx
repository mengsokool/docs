---
title: "Get Started with BeeNut: Install, Model, and Count"
sidebarTitle: "Quick Start"
description: "Install BeeNut, load a YOLO ONNX model, configure your target parts, and run your first live counting session in under 10 minutes."
---

BeeNut turns any supported machine into a dedicated object-counting kiosk. You bring a camera and a YOLO ONNX model; BeeNut handles everything else — live preview, bounding boxes, stable counts, and a touch-friendly operator interface. Follow these steps to go from a fresh machine to a working counting session in under ten minutes.

<Steps>

  <Step title="Check the prerequisites">

    Before you install, confirm you have the following:

    **Supported platform**
    - Linux desktop (AMD64) or Raspberry Pi / Debian ARM64
    - macOS (Intel or Apple Silicon)

    **Camera**
    - A USB webcam or CSI ribbon-cable camera recognized by your operating system

    **YOLO model**
    - A YOLO ONNX model file (`.onnx`) trained for the parts you want to count. BeeNut does not bundle sample weights — you supply your own.

    <Note>
      BeeNut supports two ONNX output formats: end-to-end NMS-free models (output shape `1 × N × 6`) and standard YOLOv8/v11 models (output shape `1 × channels × boxes`). Either format works without any extra configuration.
    </Note>

  </Step>

  <Step title="Install BeeNut">

    **Linux and Raspberry Pi OS — one-command installer**

    Open a terminal and run:

    ```bash
    curl -fsSL https://raw.githubusercontent.com/mengsokool/app.beenut/master/scripts/install-linux.sh | sudo bash
    ```

    The installer downloads the latest release package, verifies its `checksums.sha256` signature, and installs it with `apt`. It is idempotent — running it again upgrades or repairs an existing installation.

    By default the installer uses the `desktop` profile, which installs BeeNut as a normal application without modifying your system's boot behavior. If you want a specific profile or release version, pass environment variables before the script:

    ```bash
    curl -fsSL https://raw.githubusercontent.com/mengsokool/app.beenut/master/scripts/install-linux.sh \
      | sudo env BEENUT_VERSION=v0 BEENUT_PROFILE=appliance-pi bash
    ```

    Available install profiles:

    | Profile | What it does |
    |---|---|
    | `desktop` | Installs as a standard app; does not touch boot. Recommended for first-time setups. |
    | `appliance-linux` | Configures the machine to boot directly into the BeeNut kiosk. |
    | `appliance-pi` | Raspberry Pi kiosk using the `flutter-pi` runtime. |
    | `dev-service` | Installs the backend service only, for diagnostics or headless use. |

    <Tip>
      If you installed BeeNut from a `.deb` package manually, run `sudo beenut-setup` after installation to interactively select and apply the profile that matches your deployment scenario.
    </Tip>

    <Warning>
      The `appliance-linux` and `appliance-pi` profiles intentionally take over your machine's boot process. Your display will go straight to the BeeNut kiosk on every startup. If you end up at a black screen, SSH into the machine and run:

      ```bash
      sudo beenut-setup --recover-desktop
      ```

      `beenut-recover-desktop` is a convenience alias for the same command. Either form stops the kiosk services, re-enables your display manager, restores the normal boot target, and reboots into a desktop session.

      Do **not** use an appliance profile on a machine you still need for general-purpose work.
    </Warning>

    **macOS**

    Download the latest `.dmg` installer or `.app.zip` bundle from the [GitHub Releases page](https://github.com/mengsokool/app.beenut/releases). Open the `.dmg`, drag **BeeNut** into your Applications folder, and launch it from there.

  </Step>

  <Step title="Add your YOLO model">

    BeeNut looks for models inside the `service/models/` directory. Create a named subfolder for your model and place your files there:

    ```
    service/models/my-model/
    ├─ yolo.onnx
    └─ labels.txt   ← optional
    ```

    - **`yolo.onnx`** (required) — your exported YOLO model in ONNX format.
    - **`labels.txt`** (optional) — one class name per line, in the same order as the model's output classes.

    <Tip>
      BeeNut reads class names from the ONNX file's metadata first. Only add `labels.txt` if your ONNX file was exported without embedded class names. If neither source provides class names, BeeNut falls back to generic category labels so the app keeps running.
    </Tip>

    You can add multiple models by creating separate subfolders — for example `service/models/screws/` and `service/models/bolts/`. You switch between them in Settings without restarting.

  </Step>

  <Step title="Launch BeeNut">

    Open BeeNut from your Applications menu (macOS) or run it from the desktop (Linux). The application automatically starts the `beenutd` backend service in the background. You should see the live camera preview appear on the kiosk screen within a few seconds.

    <Note>
      If the preview does not appear, check that your camera is connected and recognized by the OS before starting BeeNut. On Linux you can confirm this with `ls /dev/video*`.
    </Note>

  </Step>

  <Step title="Select your model">

    1. Tap or click the **Settings** button in the kiosk toolbar.
    2. Open the **Model** tab.
    3. Use the file picker to select the model folder you created in Step 3 (for example, `service/models/my-model/`).
    4. Confirm your selection. BeeNut hot-reloads the model without requiring a restart.

    <Tip>
      After loading the model, check the diagnostics panel to confirm that class names are being read correctly. If you see generic labels instead of your own class names, add a `labels.txt` file to the model folder as described in Step 3.
    </Tip>

  </Step>

  <Step title="Configure your target parts">

    The **Catalog** maps the model's detected class labels to the operator-facing part names you want displayed on the kiosk.

    1. In **Settings**, open the **Catalog** tab.
    2. Tap **Add part type**.
    3. Enter a human-readable **Name** for the part (for example, "M3 Hex Nut").
    4. Enter one or more **label keywords** that match the class names from your model (for example, `nut`, `hex_nut`).
    5. Repeat for each part type you want to count.

    <Note>
      You can add as many part types as your model supports. Each part type appears as a selectable target on the main kiosk screen.
    </Note>

  </Step>

  <Step title="Start your first counting session">

    1. Return to the main kiosk screen.
    2. Use the **target selector** to pick the part type you want to count.
    3. Point the camera at your parts tray.
    4. Read the stable count displayed on screen — BeeNut applies a lower-median smoothing filter over a short sliding window of frames, so the number settles quickly and stays noise-free even when parts are still moving.

    <Tip>
      If you're using a hardware tray with a presence sensor, removing the tray instantly resets the count to zero and clears the detection history — no manual reset needed.
    </Tip>

    That's it. You're counting. Adjust the confidence threshold in **Settings → Model** if you see missed detections or false positives at high part densities.

  </Step>

</Steps>
